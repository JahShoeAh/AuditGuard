# AuditGuard Database Schema

> **Purpose:** Persistent storage for audit reports with wallet address correlation

---

## Overview

The database stores audit reports generated by autonomous agents, linked to the wallet address that deployed the smart contract. This enables users to view all audits for contracts they own.

**Deployment stack:**
- **PostgreSQL** via `DATABASE_URL` (Vercel Postgres, AWS RDS, or Supabase)
- **AWS S3** via `AWS_S3_BUCKET` — markdown content stored as `reports/{jobId}.md`
- `mdContent` is **never** stored in PostgreSQL; it is fetched from S3 on demand

---

## Data Model

```typescript
// packages/sdk/db/report-types.js (frozen shared contract)
// This interface is the canonical field set. Do NOT add/remove fields
// without coordinating all four task branches.

export interface StoredAuditReport {
  id: string;                    // "report:{jobId}"
  jobId: string;                 // On-chain AuditAuction job ID
  contractAddress: string;       // Audited contract EVM address (lowercase)
  deployerAddress: string;       // Contract deployer EVM address (lowercase)
  hederaAccountId: string | null; // Hedera 0.0.NNNN ID if available, else null
  chain: string;                 // e.g. "hedera-testnet"
  contractType: string;          // "lending"|"dex"|"staking"|"bridge"|"vault"|
                                 // "derivatives"|"oracle"|"governance"|"nft"

  // File reference (content lives in S3, never in PostgreSQL)
  s3Key: string;                 // S3 object key, e.g. "reports/42.md"
                                 // Empty string when AWS_S3_BUCKET not set (local dev)
  contentHash: string;           // SHA3-256 hex digest of markdown content
  cid: string;                   // IPFS / 0g content identifier

  // Agent metadata
  agentAddresses: string[];      // EVM addresses of winning agents
  agentCount: number;
  findingCount: number;
  findingsBySeverity: {
    critical: number;
    high: number;
    medium: number;
    low: number;
    info: number;
  };

  // Metadata
  timestamp: number;             // Unix ms when this record was written
  tags: string[];                // Keywords extracted from findings
  source: 'orchestrator' | 'agent' | 'manual';
}

// GET /api/reports/:jobId also includes mdContent fetched from S3:
// StoredAuditReport & { mdContent: string }
```

## PostgreSQL Schema

**File:** `orchestrator/src/schema.sql` (run once to initialize)

```sql
CREATE TABLE IF NOT EXISTS audit_reports (
  id                   TEXT PRIMARY KEY,
  job_id               TEXT        NOT NULL,
  contract_address     TEXT        NOT NULL,
  deployer_address     TEXT        NOT NULL,
  hedera_account_id    TEXT,
  chain                TEXT        NOT NULL,
  contract_type        TEXT        NOT NULL,
  s3_key               TEXT        NOT NULL,
  content_hash         TEXT        NOT NULL,
  cid                  TEXT,
  agent_addresses      TEXT[]      NOT NULL DEFAULT '{}',
  agent_count          INTEGER     NOT NULL DEFAULT 0,
  finding_count        INTEGER     NOT NULL DEFAULT 0,
  findings_by_severity JSONB       NOT NULL DEFAULT '{}',
  timestamp            BIGINT      NOT NULL,
  tags                 TEXT[]      NOT NULL DEFAULT '{}',
  source               TEXT        NOT NULL,
  created_at           TIMESTAMP   NOT NULL DEFAULT NOW()
);

-- Indexes for fast lookups
CREATE INDEX IF NOT EXISTS idx_deployer_address  ON audit_reports(deployer_address);
CREATE INDEX IF NOT EXISTS idx_contract_address  ON audit_reports(contract_address);
CREATE INDEX IF NOT EXISTS idx_job_id            ON audit_reports(job_id);
CREATE INDEX IF NOT EXISTS idx_timestamp         ON audit_reports(timestamp DESC);
```

**Run migration:**
```bash
# Local dev (Docker Postgres)
psql "$DATABASE_URL" -f orchestrator/src/schema.sql

# Production (run once before deploying)
DATABASE_URL=postgresql://user:pass@host:5432/auditguard psql "$DATABASE_URL" -f orchestrator/src/schema.sql
```

## S3 File Storage

Markdown content is stored separately from the database to avoid large TEXT blobs in PostgreSQL.

**Key pattern:** `reports/{jobId}.md`
**Bucket:** set by `AWS_S3_BUCKET` environment variable

```javascript
// packages/sdk/db/report-types.js
export function s3Key(jobId) {
  return `reports/${String(jobId)}.md`;
}
```

**Environment variables:**
```
AWS_S3_BUCKET         auditguard-reports
AWS_REGION            us-east-1
AWS_ACCESS_KEY_ID     (from IAM task role in ECS; from secret in local dev)
AWS_SECRET_ACCESS_KEY (from IAM task role in ECS; from secret in local dev)
```

**Local dev fallback:** When `AWS_S3_BUCKET` is not set, `s3Key` is stored as an empty string and `mdContent` is returned as an empty string from `getReportById()`. Use a local Postgres container instead.

## API Endpoints

### GET `/api/reports?deployer={address}`

Returns all reports for a specific wallet address. Does **not** include `mdContent`.

**Response:**
```json
{
  "success": true,
  "count": 1,
  "data": [
    {
      "id": "report:123",
      "jobId": "123",
      "contractAddress": "0x123...",
      "deployerAddress": "0xabc...",
      "hederaAccountId": "0.0.7951944",
      "chain": "hedera-testnet",
      "contractType": "lending",
      "s3Key": "reports/123.md",
      "contentHash": "0x...",
      "cid": "bafy...",
      "agentAddresses": ["0xagent1..."],
      "agentCount": 3,
      "findingCount": 15,
      "findingsBySeverity": { "critical": 2, "high": 5, "medium": 6, "low": 2, "info": 0 },
      "timestamp": 1740000000000,

      "source": "orchestrator"
    }
  ]

```

---

### GET `/api/reports/:jobId`

Returns the specific report **plus** `mdContent` fetched from S3.

**Response:**
```json
{
  "success": true,
  "data": {
    "id": "report:123",
    "s3Key": "reports/123.md",
    "contentHash": "0x...",
    "cid": "bafy...",
    "mdContent": "# Audit Report...\n"
  }
}
```

---

## Deployment Checklist

1. Create PostgreSQL database (Vercel Postgres / AWS RDS / Supabase)
2. Run `psql "$DATABASE_URL" -f orchestrator/src/schema.sql`
3. Create S3 bucket `auditguard-reports` in AWS
4. Attach ECS task role with `s3:PutObject` + `s3:GetObject` on `arn:aws:s3:::auditguard-reports/*`
5. Set `DATABASE_URL`, `AWS_S3_BUCKET`, `AWS_REGION`, `CORS_ORIGIN`, `API_PORT` in ECS task definition
6. Set `VITE_API_BASE_URL=https://api.your-domain.com` in Vercel project settings
7. Deploy orchestrator + API server as Docker containers on ECS
8. Deploy dashboard to Vercel
